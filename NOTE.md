# PD 循線調參數筆記

能動的三個旋鈕（line_follower.ino 最上面）：
- kp = 100          比例增益：離線越遠，修正越用力
- kd = 200          微分增益：抑制左右擺動（阻尼）
- Basic_speed = 100 基礎速度：整體多快

## 調參數標準流程（一次只動一個）
1. 速度先定低（Basic_speed 先 80~100），慢慢調比較安全
2. Kd 先設 0，只調 Kp：從小往上加，直到能跟線但直線會 S 形左右擺 → 代表 Kp 夠力
3. 再加 Kd：慢慢加，把左右擺震盪壓下來，直到直線平順、過彎又不鈍
4. 最後才拉速度：Basic_speed 往上加。速度變快後 Kp/Kd 通常要一起再往上，再回步驟 2、3 微調

經驗法則：Kd 大約是 Kp 的 2~5 倍（現在 100/200 = 2 倍，合理起點）

## 看症狀反推要改誰
| 車子的行為 | 原因 | 怎麼改 |
|-----------|------|--------|
| 直線一直左右擺（S 形） | Kp 太高 或 Kd 不夠 | 降 Kp，或加 Kd |
| 過彎切太進去 / 飛出賽道 | Kp 太低 或 速度太快 | 加 Kp，或降速度 |
| 反應慢吞吞、跟不上彎 | Kp 太低 | 加 Kp |
| 抖動、抽搐、很神經質 | Kd 太高（放大感測器雜訊） | 降 Kd |
| 慢的時候好好的，一快就爆走 | 速度跟 Kp/Kd 不匹配 | 拉速度時 Kp、Kd 一起加 |

## 下一步建議
先讓車子「說話」再調，不要盲調：
- Print1() 的序列埠輸出目前全被註解掉，打開後印出 x（賽道位置）/ err（誤差）/ pd，
  邊跑邊看序列埠繪圖器就能判斷是擺動還是遲鈍，調起來快很多
- 或把 kp/kd/Basic_speed 改成用按鈕現場微調，不用每次重新編譯上傳
